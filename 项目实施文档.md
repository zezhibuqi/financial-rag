### 操作步骤

#### 1.1 Supabase SQL 执行（在 Supabase Dashboard 的 SQL Editor 中）

```sql
-- 启用向量扩展
create extension if not exists vector;

-- 创建主表（注意维度是 1024，不是 1536）
create table chunks (
  id serial primary key,
  doc_name text not null,
  company text not null,        -- 公司简称："茅台" / "宁德时代"
  year int not null,
  page int not null,
  chunk_type text,
  content text not null,
  embedding vector(1024),
  uploaded_at timestamptz default now()
);

-- 创建索引加速检索
create index on chunks using ivfflat (embedding vector_cosine_ops) with (lists = 100);

-- 关闭 RLS（简化演示）
alter table chunks disable row level security;

-- 创建匹配函数（供后端 RPC 调用）
create or replace function match_chunks(
  query_embedding vector(1024),
  match_count int,
  filter_company text default null,
  filter_year int default null
)
returns table(
  id int,
  content text,
  doc_name text,
  page int,
  similarity float
)
language sql stable
as $$
  select
    id,
    content,
    doc_name,
    page,
    1 - (embedding <=> query_embedding) as similarity
  from chunks
  where
    (filter_company is null or company = filter_company) and
    (filter_year is null or year = filter_year)
  order by embedding <=> query_embedding
  limit match_count;
$$;
```

#### 1.2 编写离线灌库脚本 backend/ingest.py

> 基于对贵州茅台（143页）和宁德时代（232页）年报的实际解析结果。

**步骤一：公司简称映射**

```python
# 硬编码映射表：文件名中提取的公司名 → 简称
COMPANY_MAPPING = {
    "贵州茅台": "茅台",
    "宁德时代": "宁德时代",
}

import re, os

def parse_metadata(filename):
    """从文件名提取公司和年份，映射为简称"""
    m = re.match(r'(.+?)(\d{4})年?度?报告', filename)
    if m:
        raw_company, year = m.group(1), int(m.group(2))
        company = COMPANY_MAPPING.get(raw_company, raw_company)
        return company, year
    return filename, None
```

**步骤二：PDF 读取与页面遍历**

```python
import pdfplumber
pdf = pdfplumber.open(pdf_path)
for page_num, page in enumerate(pdf.pages, start=1):
    text = page.extract_text() or ""
    tables = page.extract_tables() or []
```

**步骤三：页眉页脚清洗**

两份年报的页眉页脚模式不同，需分别处理：

| 年报 | 页眉（每页第一行） | 页脚（每页最后一行） |
|------|---------------------|----------------------|
| 茅台 | `贵州茅台酒股份有限公司2025年年度报告` | `N/143`（如 `6/143`） |
| 宁德时代 | `宁德时代新能源科技股份有限公司2025年年度报告全文` | 纯页码数字（如 `11`、`116`） |

清洗逻辑：

```python
import re

def clean_header_footer(text, company_keywords):
    """去除每页的页眉和页脚"""
    lines = text.strip().split('\n')
    # 去除页眉：第一行包含公司名+年度报告
    if lines and any(kw in lines[0] for kw in company_keywords):
        lines = lines[1:]
    # 去除页脚：最后一行是纯页码（"N/总数"或纯数字）
    if lines and re.match(r'^\d+(/\d+)?$', lines[-1].strip()):
        lines = lines[:-1]
    return '\n'.join(lines)
```

`company_keywords` 由第一页自动检测（第一页必含公司全称）。

**步骤四：文本与表格分离处理**

每页同时包含自由文本和表格。关键原则：**表格区域的文本不要重复计入**。

```python
def extract_page_content(page):
    """提取一页的文本和表格，避免重复"""
    tables = page.extract_tables() or []
    text = page.extract_text() or ""
    text = clean_header_footer(text, company_keywords)
    return text, tables
```

**步骤五：表格清洗（关键难点）**

```python
def clean_table(table):
    """清洗 pdfplumber 提取的表格"""
    if not table or len(table) < 2:
        return None

    # 1. 去除全 None 的列（分隔列）
    num_cols = max(len(row) for row in table)
    non_none_cols = [
        c for c in range(num_cols)
        if any(row[c] is not None for row in table if c < len(row))
    ]
    table = [
        [row[c] if c < len(row) else None for c in non_none_cols]
        for row in table
    ]

    # 2. 合并被拆分的行
    merged = [table[0]]
    for row in table[1:]:
        if row[0] is None and len(merged) > 0:
            prev = merged[-1]
            for i, cell in enumerate(row):
                if cell and i < len(prev):
                    prev[i] = (prev[i] or '') + cell
        else:
            merged.append(row)
    table = merged

    # 3. 将 None 替换为空字符串，清理换行符
    table = [
        [(cell or '').replace('\n', ' ').strip() for cell in row]
        for row in table
    ]

    # 4. 去除全空行
    table = [row for row in table if any(cell for cell in row)]

    return table
```

**步骤六：表格转 Markdown**

```python
def table_to_markdown(table):
    """将清洗后的表格转为 Markdown 格式"""
    if not table or len(table) < 2:
        return ""

    max_cols = max(len(row) for row in table)
    rows_padded = [row + [''] * (max_cols - len(row)) for row in table]

    lines = []
    lines.append('| ' + ' | '.join(rows_padded[0]) + ' |')
    lines.append('| ' + ' | '.join(['---'] * max_cols) + ' |')
    for row in rows_padded[1:]:
        lines.append('| ' + ' | '.join(row) + ' |')

    return '\n'.join(lines)
```

若 Markdown 超过 1024 字符，按行切半（保留表头），每半部分作为一个独立 chunk。

**步骤七：文本切片**

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=512,         # 字符数
    chunk_overlap=50,       # 字符数
    separators=['\n\n', '\n', '。', '；', '，', ' ']
)
text_chunks = splitter.split_text(cleaned_text)
```

> 注：`chunk_size=512` 和 `chunk_overlap=50` 单位均为**字符数**，不是 token。

**步骤八：Embedding 与入库**

```python
from sentence_transformers import SentenceTransformer

# 硬编码本地模型路径，避免运行时联网
model = SentenceTransformer(r'C:\Users\用户名\.cache\modelscope\AI-ModelScope\bge-large-zh-v1.5')

# 灌库幂等：先删除同名年报的旧 Chunk
supabase.table('chunks').delete().eq('doc_name', filename).execute()

# 批量编码（GPU加速）
all_chunks = []  # [(content, chunk_type, page_num), ...]
texts = [c[0] for c in all_chunks]
embeddings = model.encode(texts, batch_size=64, show_progress_bar=True)

# 批量写入 Supabase（每次 100 条，避免超时）
for i in range(0, len(all_chunks), 100):
    batch = [
        {
            "doc_name": filename,
            "company": company,
            "year": year,
            "page": all_chunks[j][2],
            "chunk_type": all_chunks[j][1],
            "content": all_chunks[j][0],
            "embedding": embeddings[j].tolist()
        }
        for j in range(i, min(i + 100, len(all_chunks)))
    ]
    supabase.table("chunks").insert(batch).execute()
```

**实际解析中的坑（务必注意）**

1. **茅台的表格表头有多行**：如"本期比上年同期增减(%)"在表头中跨 3 行显示。`clean_table` 中的 None 列过滤和行合并逻辑会自动处理，但转 Markdown 后表头可能不够语义化。对 RAG 来说够用——LLM 能从上下文理解列含义。

2. **宁德时代的利润表跨多页**：每个 chunk 独立入库即可，LLM 能从片段中识别上下文。

3. **金额单位不一致**：茅台全文用"元"，宁德时代部分报表用"千元"。表格转 Markdown 时，建议在 chunk 内容前附加页面上的单位说明行（如"单位：元 币种：人民币"），方便 LLM 正确理解数量级。

4. **表格与正文混排**：部分页面同时有多个表格和正文。文本 chunk 和表格 chunk 分别入库，page 字段相同，检索时都能命中。

#### 2.3 实现 /api/upload

@app.route('/api/upload', methods=['POST'])
def upload():
    file = request.files.get('file')
    if not file:
        return jsonify({"error": "请选择要上传的文件"})

    # 上传至 Supabase Storage
    supabase.storage.from_('pdfs').upload(

        file.read(),

#### 2.4 实现 /api/docs

查询 chunks 表，按 `doc_name` 去重聚合，同时返回 distinct company 和 year 用于前端下拉框：

```python
@app.route('/api/docs', methods=['GET'])
def docs():
    result = supabase.table('chunks').select('doc_name,company,year,uploaded_at').execute()

    # distinct company / year
    companies = sorted(set(r['company'] for r in result.data))
    years = sorted(set(r['year'] for r in result.data))

    # 按 doc_name 聚合
    docs_map = {}
    for r in result.data:
        key = r['doc_name']
        if key not in docs_map:
            docs_map[key] = {
                "doc_name": r['doc_name'],
                "company": r['company'],
                "year": r['year'],
                "chunk_count": 0,
                "uploaded_at": r['uploaded_at']
            }
        docs_map[key]['chunk_count'] += 1

    return jsonify({
        "companies": companies,
        "years": years,
        "docs": list(docs_map.values())
    })
```

## 🎨 阶段 3：前端界面与前后端联调

### 目标

构建三个页面（问答页、管理页、关于页），打通前端与后端 API 的调用，实现完整的交互体验。

### 操作步骤

#### 3.1 配置 Next.js 代理（解决开发环境跨域）

在 `frontend/next.config.js` 中添加：

```js
module.exports = {
  rewrites: async () => [
    { source: '/api/:path*', destination: 'http://localhost:5000/api/:path*' },
  ],
};
```

#### 3.2 实现问答页 (pages/index.tsx)

组件结构：
- `Select`（公司选择，默认"不限"） + `Select`（年份选择，默认"不限"）
- 选项数据从 `GET /api/docs` 动态获取
- `Input.Search` 作为提问框
- 空态：回答区和 sources 区隐藏，提问框在页面上方
- 加载态：回答区显示 `Spin`
- 结果态：`Card` 展示回答 + `Collapse.Panel` 展示 sources（文件名、页码、片段预览、相似度）
- 错误态：`message.error()` 弹窗提示

#### 3.3 实现管理页 (pages/manage.tsx)

- `Upload` 组件（`action="/api/upload"`，Bucket 名 `pdfs`）
- `Table` 展示已入库年报列表（调用 `/api/docs` 的 `docs` 字段），列：公司、年份、Chunk 数量、入库时间
- 上传成功后 `message.success()`，Table 需手动刷新（不自动轮询）

#### 3.4 实现关于页 (pages/about.tsx)

静态页面，列出技术栈和项目简介（Next.js + Flask + Supabase + DeepSeek）。

### ✅ 阶段验证

- 前端 `npm run dev` 无报错，三个页面路由切换正常
- 在问答页输入"茅台2023毛利率"，先选择"茅台"和"2023"，点击搜索后页面显示出回答并自动展开 citation 面板
- 不选公司/年份直接提问"宁德时代和茅台 2023 年营收增速谁更高？"，回答准确
- 管理页能显示已入库的 PDF 列表，公司、年份、Chunk 数量正确
- 上传一个小文件（<1MB），页面提示成功，且 Storage 中能看到文件

## 🚀 阶段 4：部署上线与验收测试

### 目标

将前端部署至 Vercel，后端部署至 Vercel Serverless（Embedding 走远程 API，无需加载模型），并执行最终的 Golden Set 测试。

### 操作步骤

#### 4.1 前端部署（Vercel）

```bash
cd frontend
# 设置环境变量
vercel env add NEXT_PUBLIC_API_URL
# 输入后端地址，如 https://your-backend.vercel.app
npm run build  # 本地先验证构建无错
vercel --prod
```

#### 4.2 后端部署（Vercel Serverless）

后端不需要加载 `sentence-transformers` 模型（Embedding 已改用远程 API），因此包体积远低于 50MB 限制，可正常部署至 Vercel Serverless。

在 `backend` 目录下创建 `vercel.json`：

```json
{
  "functions": {
    "api/*.py": { "maxDuration": 10 }
  },
  "rewrites": [{ "source": "/(.*)", "destination": "/api/app" }]
}
```

将 `app.py` 重命名为 `api/app.py`（适配 Vercel 规则），运行 `vercel --prod`。

#### 4.3 配置 Supabase Storage 公开访问

确保 Bucket `pdfs` 为公开可读（用于文件下载链接），或至少对后端 Service Role 可访问。

#### 4.4 最终验收（Golden Set）

在线上环境（公网 URL）测试以下 3 题：

| # | 问题 | 期望 |
|---|------|------|
| 1 | "茅台 2023 毛利率是多少？" | 返回具体数字（如 91.96%），sources 指向正确页码 |
| 2 | "宁德时代 2024 营收增速是多少？" | 返回具体数字，sources 指向正确页码 |
| 3 | "宁德时代和茅台 2023 年营收增速谁更高？" | 排序正确，跨公司比较准确 |

### ✅ 阶段验证

- 3 道 Golden Set 全部通过（回答准确 + sources 指向正确）
- Vercel 后台无函数超时或内存溢出报错
- 管理页上传功能正常，文件出现在 Supabase Storage 中
