### 操作步骤

#### 1.1 Supabase SQL 执行（在 Supabase Dashboard 的 SQL Editor 中）

```sql
-- 启用向量扩展
create extension if not exists vector;

-- 创建主表（注意维度是 1024，不是 1536）
create table chunks (
  id serial primary key,
  doc_name text not null,
  company text not null,
  year int not null,
  page int not null,
  chunk_type text,
  content text not null,
  embedding vector(1024),
  uploaded_at timestamptz default now()
);

-- 创建索引加速检索
create index on chunks using ivfflat (embedding vector_cosine_ops) with (lists = 100);

-- 关闭 RLS（简化演示）
alter table chunks disable row level security;

-- 创建匹配函数（供后端 RPC 调用）
create or replace function match_chunks(
  query_embedding vector(1024),
  match_count int,
  filter_company text default null,
  filter_year int default null
)
returns table(
  id int,
  content text,
  doc_name text,
  page int,
  similarity float
)
language sql stable
as $$
  select
    id,
    content,
    doc_name,
    page,
    1 - (embedding <=> query_embedding) as similarity
  from chunks
  where
    (filter_company is null or company = filter_company) and
    (filter_year is null or year = filter_year)
  order by embedding <=> query_embedding
  limit match_count;
$$;
```

**步骤二：页眉页脚清洗**

两份年报的页眉页脚模式不同，需分别处理：

| 年报 | 页眉（每页第一行） | 页脚（每页最后一行） |
|------|---------------------|----------------------|
| 茅台 | `贵州茅台酒股份有限公司2025年年度报告` | `N/143`（如 `6/143`） |
| 宁德时代 | `宁德时代新能源科技股份有限公司2025年年度报告全文` | 纯页码数字（如 `11`、`116`） |

清洗逻辑：

```python
import re

def clean_header_footer(text, company_keywords):
    """去除每页的页眉和页脚"""
    lines = text.strip().split('\n')
    # 去除页眉：第一行包含公司名+年度报告
    if lines and any(kw in lines[0] for kw in company_keywords):
        lines = lines[1:]
    # 去除页脚：最后一行是纯页码（"N/总数"或纯数字）
    if lines and re.match(r'^\d+(/\d+)?$', lines[-1].strip()):
        lines = lines[:-1]
    # 去除茅台特有的倒数第二行章节导航（如"第一节 目录"、"□适用 √不适用"）
    return '\n'.join(lines)
```

`company_keywords` 按文件自动检测：从第一页提取公司简称，后续页复用。

**步骤三：文本与表格分离处理**

每页同时包含自由文本和表格。关键原则：**表格区域的文本不要重复计入**。

```python
def extract_page_content(page):
    """提取一页的文本和表格，避免重复"""
    # 1. 提取表格并记录其在页面中的位置
    tables = page.extract_tables() or []
    
    # 2. 提取纯文本（已去除表格区域的重复文本）
    #    pdfplumber 的 extract_text() 会包含表格内容，但格式混乱
    #    策略：文本和表格分别处理，文本chunk标记chunk_type='text'，表格标记'table'
    text = page.extract_text() or ""
    text = clean_header_footer(text, company_keywords)
    
    return text, tables
```

**步骤四：表格清洗（关键难点）**

实际分析发现两个核心问题：

**问题 A — 合并单元格产生大量 None**。宁德时代的财务报表（资产负债表/利润表/现金流量表）是 9 列宽表，其中第 0、2、4、6、8 列几乎全是 None（用于视觉分隔），None 占比高达 40-50%。茅台的多行表头也有类似问题。

**问题 B — 长文本跨行拆分**。宁德时代利润表中"信用减值损失（损失以'-'号填列）"被拆成两行：第一行 `信用减值损失（损失以"-"号填`，第二行 `列）`。

清洗逻辑：

```python
def clean_table(table):
    """清洗 pdfplumber 提取的表格"""
    if not table or len(table) < 2:
        return None
    
    # 1. 去除全 None 的列（分隔列）
    num_cols = max(len(row) for row in table)
    non_none_cols = [
        c for c in range(num_cols)
        if any(row[c] is not None for row in table if c < len(row))
    ]
    table = [
        [row[c] if c < len(row) else None for c in non_none_cols]
        for row in table
    ]
    
    # 2. 合并被拆分的行：如果某行第一列为 None 且上一行第一列非 None，
    #    则将该行内容追加到上一行（处理长文本跨行）
    merged = [table[0]]
    for row in table[1:]:
        if row[0] is None and len(merged) > 0:
            # 将非None值追加到上一行对应列
            prev = merged[-1]
            for i, cell in enumerate(row):
                if cell and i < len(prev):
                    prev[i] = (prev[i] or '') + cell
        else:
            merged.append(row)
    table = merged
    
    # 3. 将 None 替换为空字符串，清理换行符
    table = [
        [(cell or '').replace('\n', ' ').strip() for cell in row]
        for row in table
    ]
    
    # 4. 去除全空行
    table = [row for row in table if any(cell for cell in row)]
    
    return table
```

**步骤五：表格转 Markdown**

```python
def table_to_markdown(table):
    """将清洗后的表格转为 Markdown 格式"""
    if not table or len(table) < 2:
        return ""
    
    # 取第一行作为表头
    header = table[0]
    # 对齐列数
    max_cols = max(len(row) for row in table)
    rows_padded = [row + [''] * (max_cols - len(row)) for row in table]
    
    lines = []
    lines.append('| ' + ' | '.join(rows_padded[0]) + ' |')
    lines.append('| ' + ' | '.join(['---'] * max_cols) + ' |')
    for row in rows_padded[1:]:
        lines.append('| ' + ' | '.join(row) + ' |')
    
    return '\n'.join(lines)
```

若 Markdown 超过 1024 tokens，按行切半（保留表头），每半部分作为一个独立 chunk。

**步骤七：元数据提取**

从文件名自动推断 `company` 和 `year`：

```python
import re, os

def parse_metadata(filename):
    """从文件名提取公司和年份"""
    # 支持格式：贵州茅台2025年度报告.pdf、宁德时代2025年度报告.pdf
    m = re.match(r'(.+?)(\d{4})年?度?报告', filename)
    if m:
        return m.group(1), int(m.group(2))
    return filename, None

company, year = parse_metadata(os.path.basename(pdf_path))
```

**步骤八：Embedding 与入库**

```python
from sentence_transformers import SentenceTransformer
model = SentenceTransformer('BAAI/bge-large-zh-v1.5')

# 批量编码（GPU加速）
all_chunks = []  # [(content, chunk_type, page_num), ...]
# ... 填充 all_chunks ...

texts = [c[0] for c in all_chunks]
embeddings = model.encode(texts, batch_size=64, show_progress_bar=True)

# 批量写入 Supabase（每次 100 条，避免超时）
for i in range(0, len(all_chunks), 100):
    batch = [
        {
            "doc_name": filename,
            "company": company,
            "year": year,
            "page": all_chunks[j][2],
            "chunk_type": all_chunks[j][1],
            "content": all_chunks[j][0],
            "embedding": embeddings[j].tolist()
        }
        for j in range(i, min(i + 100, len(all_chunks)))
    ]
    supabase.table("chunks").insert(batch).execute()
```

**⚠️ 实际解析中的坑（务必注意）**

1. **茅台的表格表头有多行**：如"本期比上年同期增减(%)"在表头中跨 3 行显示。`clean_table` 中的 None 列过滤和行合并逻辑会自动处理，但转 Markdown 后表头可能不够语义化。对 RAG 来说够用——LLM 能从上下文理解列含义。

2. **宁德时代的利润表跨多页**：合并利润表从第 116 页延续到第 118 页（35+49+6 行），母公司利润表从第 118 页到第 119 页。每个 chunk 独立入库即可，LLM 能从片段中识别上下文。

3. **金额单位不一致**：茅台全文用"元"（如 `168,838,102,514.79`），宁德时代部分报表用"千元"（如 `423,701,834` 表示约 4237 亿元）。表格转 Markdown 时，建议在 chunk 内容前附加页面上方的单位说明行（如"单位：元 币种：人民币"），方便 LLM 正确理解数量级。

4. **表格与正文混排**：部分页面（如茅台第 6 页）同时有"主要会计数据"和"主要财务指标"两个表格，中间夹着少量正文。文本 chunk 和表格 chunk 分别入库，page 字段相同，检索时都能命中。
