# ADR-0001: 上传与灌库分离

## 状态

已接受

## 日期

2026-07-01

## 上下文

系统提供两个入口让用户提供年报 PDF：
1. 前端管理页通过 `/api/upload` 上传
2. 管理员在本地运行 `ingest.py` 离线灌库

将两者串联（上传后自动解析入库）是大多数 RAG 系统的默认行为。但本项目后端部署在 Vercel Serverless 上，存在 10 秒函数超时限制。PDF 解析 + Embedding 在包含复杂表格（如宁德时代 9 列合并财务报表）的年报上耗时远超 10 秒。

## 决策

上传（`/api/upload`）仅将 PDF 存至 Supabase Storage，不触发任何解析。灌库（`ingest.py`）作为独立脚本在本地 GPU 环境离线执行，由开发者或部署者手动操作。

## 后果

### 优势
- 完全规避 Vercel Serverless 超时风险
- 灌库利用本地 RTX 4060 GPU 加速，速度远高于云端 CPU
- 上传接口响应时间 < 200ms，稳定可靠
- 职责清晰：上传 = 暂存，灌库 = 索引

### 劣势
- 用户上传 PDF 后不会立即出现在检索结果中，需等待人工灌库
- 对非技术人员不友好，无法自动化
- 管理页展示的已入库列表可能滞后于上传进度

### 缓解措施
- 这是一个面向答辩演示的项目，年报数据已提前灌好（6 份），上传功能仅作形式展示
- 管理页 Table 仅展示 chunks 表中的已入库数据，Upload 组件独立并存，用户不会产生"上传即可用"的错觉
