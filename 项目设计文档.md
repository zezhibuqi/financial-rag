# 项目设计文档

**项目名称**：财报智能问答系统（RAG）

## 1. 系统总体架构

```text
┌──────────────────────────────────────────────────────────────┐
│ 前端（Next.js 14 Pages Router + Ant Design）                 │
│  ├─ 问答页面（提问 + 公司/年份下拉框 + Markdown答案 + 可展开来源）│
│  ├─ 财报管理页面（上传 + 已入库列表）                           │
│  └─ 关于页面                                                  │
├──────────────────────────────────────────────────────────────┤
│ 后端（Python Flask）                                          │
│  ├─ /api/chat   → RAG主链路（向量检索 + 关键词补充 + Rerank   │
│  │                  + DeepSeek生成，一次性返回）               │
│  ├─ /api/upload → PDF上传（仅存Storage，解析在本地离线执行）   │
│  └─ /api/docs   → 已入库年报列表 + 筛选选项                    │
├──────────────────────────────────────────────────────────────┤
│ 数据层（Supabase PostgreSQL + pgvector）                      │
│  ├─ chunks表（id, doc_name, company, year, page,             │
│  │            chunk_type, content, embedding vector(1024),    │
│  │            uploaded_at）                                   │
│  ├─ Storage Bucket `pdfs`（存储原始PDF）                      │
│  └─ Auth + RLS（仅用于大纲要求，chunks表关闭RLS）             │
├──────────────────────────────────────────────────────────────┤
│ RAG管线                                                       │
│  ├─ 解析：pdfplumber（提取text + tables）                    │
│  ├─ 切片：正文512字符（重叠50字符），表格整表（≤1024字符）     │
│  ├─ 检索：pgvector cosine k=10 + 关键词LIKE补充              │
│  ├─ Rerank：SiliconFlow bge-reranker-v2-m3（免费API）        │
│  │         向量 top-4 + 关键词 per-company best-2 = 6~8条     │
│  ├─ Embedding（离线）：BAAI/bge-large-zh-v1.5（本地GPU）     │
│  ├─ Embedding（在线）：SiliconFlow远程API（同一模型，1024维） │
│  └─ 生成：DeepSeek API（deepseek-v4-flash）                   │
└──────────────────────────────────────────────────────────────┘
```

## 2. 技术选型决策表

| 层级 | 选型 | 替代方案 | 理由 |
|------|------|----------|------|
| 前端 | Next.js 14 + Ant Design + react-markdown | Streamlit/Gradio | AntD企业级组件；Markdown渲染回答效果 |
| 后端 | Flask | FastAPI | 大纲D3授课内容，无需另学 |
| 数据库 | Supabase (pgvector) | Chroma/Milvus | 大纲指定BaaS；pgvector开箱即用 |
| Embedding（离线） | BAAI/bge-large-zh-v1.5 (1024维) | OpenAI embedding | 中文财报友好，本地GPU可跑，0费用 |
| Embedding（在线） | SiliconFlow远程API（同模型） | 本地加载 | 避免Vercel Serverless 50MB包大小限制 |
| Reranker | BAAI/bge-reranker-v2-m3 (SiliconFlow) | Cohere/本地部署 | 免费API，中文财务语义匹配好 |
| LLM | DeepSeek API | 混元/GPT-4o | 中文理解好，上下文窗口1M |
| PDF解析 | pdfplumber | unstructured/LlamaParse | 轻量，表格抽取对财报够用 |
| 文本切片 | LangChain RecursiveCharacterTextSplitter | 手写滑动窗口 | 成熟稳定，支持递归分隔符 |

## 4. API接口规范（Flask）

### 4.1 POST /api/chat

请求体：

```json
{
  "question": "茅台2023年毛利率是多少？",
  "company": "茅台",
  "year": 2023
}
```

> `company` 和 `year` 为可选字段。若未提供，从问题文本中自动提取公司名和年份。

成功响应：

```json
{
  "answer": "根据2023年年报，茅台酒类业务毛利率为 **92.11%** ...",
  "sources": [
    {
      "doc_name": "贵州茅台2023年度报告.pdf",
      "page": 14,
      "content": "全文chunk内容...",
      "snippet": "前200字符摘要...",
      "similarity": 0.9924,
      "source": "vector"
    }
  ]
}
```

> `similarity` 为 Rerank 后的 `relevance_score`（0~1）。`source` 标记来源：`vector`（向量检索）或 `keyword`（关键词补充）。

错误响应：

```json
{"error": "问题不能为空"}
```

## 5. RAG管线详细设计

### 5.1 离线预处理（不变）

### 5.2 在线问答流程（/api/chat）

```
Query
│
├─ 1. Embedding（SiliconFlow API）
│     question → 1024维向量
│
├─ 2a. 向量检索
│     match_chunks(k=10, company?, year?)
│
├─ 2b. 关键词补充（通用财务指标）
│     触发词映射：利润→净利润/利润总额、现金流→经营活动现金流量、
│     毛利率→毛利率、分红→现金分红/派发现金红利 等
│     从问题提取所有年份、按公司名过滤、拍平去重、上限6条
│
├─ 3. 合并去重 → Rerank
│     全量 chunk（向量 + 关键词）→ bge-reranker-v2-m3 API
│     → 向量 top-4 + 关键词 per-company best-2 = 6~8条
│
├─ 4. Prompt 组装
│     每 chunk 标注 [公司 年报 第N页]
│     提示 LLM 提取数据作答，无数据时才说"未在资料中找到"
│
└─ 5. DeepSeek 生成 → {answer, sources}
```

### 5.3 错误处理

| 环节 | 错误描述 |
|------|---------|
| Embedding API | "Embedding服务不可用，请稍后重试" |
| Supabase RPC | "数据库查询失败，请稍后重试" |
| Rerank API（降级） | 失败时不阻塞，降级为 similarity 排序取 top-6 |
| LLM API | "AI服务暂时不可用，请稍后重试" |
| question 为空 | "问题不能为空" |
| /api/upload 文件缺失 | "请选择要上传的文件" |
