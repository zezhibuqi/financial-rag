# 项目设计文档

**项目名称**：财报智能问答系统（RAG）

## 1. 系统总体架构

```text
┌──────────────────────────────────────────────────────────────┐
│ 前端（Next.js 14 Pages Router + Ant Design）                 │
│  ├─ 问答页面（提问 + 公司/年份下拉框 + 答案 + citation折叠面板）│
│  ├─ 财报管理页面（上传 + 已入库列表）                           │
│  └─ 关于页面                                                  │
├──────────────────────────────────────────────────────────────┤
│ 后端（Python Flask）                                          │
│  ├─ /api/chat   → RAG主链路（一次性返回，非流式）              │
│  ├─ /api/upload → PDF上传（仅存Storage，解析在本地离线执行）   │
│  └─ /api/docs   → 已入库年报列表 + 筛选选项                    │
├──────────────────────────────────────────────────────────────┤
│ 数据层（Supabase PostgreSQL + pgvector）                      │
│  ├─ chunks表（id, doc_name, company, year, page,             │
│  │            chunk_type, content, embedding vector(1024),    │
│  │            uploaded_at）                                   │
│  ├─ Storage Bucket `pdfs`（存储原始PDF）                      │
│  └─ Auth + RLS（仅用于大纲要求，chunks表关闭RLS）             │
├──────────────────────────────────────────────────────────────┤
│ RAG管线（离线预处理 + 在线检索生成）                           │
│  ├─ 解析：pdfplumber（提取text + tables）                    │
│  ├─ 切片：正文512字符（重叠50字符），表格整表（≤1024字符）     │
│  ├─ Embedding（离线）：BAAI/bge-large-zh-v1.5（本地GPU）     │
│  ├─ Embedding（在线）：SiliconFlow远程API（同一模型，1024维） │
│  ├─ 检索：pgvector cosine，k=4，带company/year过滤            │
│  └─ 生成：DeepSeek API（兼容OpenAI SDK）                      │
└──────────────────────────────────────────────────────────────┘
```

## 2. 技术选型决策表

| 层级 | 选型 | 替代方案 | 理由 |
|------|------|----------|------|
| 前端 | Next.js 14 + Ant Design | Streamlit/Gradio | 大纲指定；AntD企业级组件，AI友好，Trae生成快 |
| 后端 | Flask | FastAPI | 大纲D3授课内容，无需另学 |
| 数据库 | Supabase (pgvector) | Chroma/Milvus | 大纲指定BaaS；pgvector开箱即用，无需额外向量库服务 |
| Embedding（离线） | BAAI/bge-large-zh-v1.5 (1024维) | OpenAI embedding | 中文财报友好，本地GPU可跑，0费用 |
| Embedding（在线） | SiliconFlow远程API（同模型） | 本地加载 | 避免Vercel Serverless 50MB包大小限制，线上无需加载模型 |
| LLM | DeepSeek API（兼容OpenAI SDK） | 混元/GPT-4o | 中文理解好，上下文窗口1M，学生免费额度够用 |
| PDF解析 | pdfplumber | unstructured/LlamaParse | 轻量，表格抽取对财报够用 |
| 文本切片 | LangChain RecursiveCharacterTextSplitter | 手写滑动窗口 | 成熟稳定，支持递归分隔符，需安装 `langchain-text-splitters` |
| 部署 | 前端Vercel，后端Vercel（Embedding走远程API） | 云服务器 | 演示保命方案：upload仅存Storage，解析离线本地跑 |

## 3. 数据库设计（Supabase）

### 3.1 启用pgvector扩展

```sql
create extension if not exists vector;
```

### 3.2 主表 chunks

```sql
create table chunks (
  id serial primary key,
  doc_name text not null,          -- 如 "贵州茅台2023年度报告.pdf"
  company text not null,           -- 公司简称："茅台" / "宁德时代"
  year int not null,               -- 2023 / 2024 / 2025
  page int not null,
  chunk_type text,                 -- 'text' 或 'table'
  content text not null,           -- 切片原文（或Markdown表格）
  embedding vector(1024),          -- bge-large-zh 输出维度
  uploaded_at timestamptz default now()  -- 入库时间戳
);

-- 为加速检索创建索引（余弦距离）
create index on chunks using ivfflat (embedding vector_cosine_ops) with (lists = 100);
```

### 3.3 关闭RLS（简化演示）

```sql
alter table chunks disable row level security;
```

### 3.4 创建匹配函数（供后端RPC调用）

```sql
create or replace function match_chunks(
  query_embedding vector(1024),
  match_count int,
  filter_company text default null,
  filter_year int default null
)
returns table(
  id int,
  content text,
  doc_name text,
  page int,
  similarity float
)
language sql stable
as $$
  select
    id,
    content,
    doc_name,
    page,
    1 - (embedding <=> query_embedding) as similarity
  from chunks
  where
    (filter_company is null or company = filter_company) and
    (filter_year is null or year = filter_year)
  order by embedding <=> query_embedding
  limit match_count;
$$;
```

## 4. API接口规范（Flask）

### 4.1 POST /api/chat

请求体：

```json
{
  "question": "茅台2023年毛利率是多少？",
  "company": "茅台",
  "year": 2023
}
```

> 注：`company` 和 `year` 为可选字段，若未提供则不过滤（走全库检索）。

成功响应：

```json
{
  "answer": "根据2023年年报，茅台毛利率为91.96%...",
  "sources": [
    {
      "doc_name": "贵州茅台2023年度报告.pdf",
      "page": 45,
      "snippet": "利润表显示营业收入...",
      "similarity": 0.92
    }
  ]
}
```

错误响应：

```json
{
  "error": "Embedding服务不可用，请稍后重试"
}
```

### 4.2 POST /api/upload

请求：`multipart/form-data`，字段 `file`

响应：

```json
{
  "msg": "文件已上传，管理员将在本地完成解析入库",
  "file_id": "xxx"
}
```

实现：仅保存至 Supabase Storage Bucket `pdfs`，不触发任何解析（避免 Vercel 超时）。

### 4.3 GET /api/docs

响应：

```json
{
  "companies": ["茅台", "宁德时代"],
  "years": [2023, 2024, 2025],
  "docs": [
    {
      "doc_name": "贵州茅台2023年度报告.pdf",
      "company": "茅台",
      "year": 2023,
      "chunk_count": 245,
      "uploaded_at": "2026-06-25T10:00:00Z"
    }
  ]
}
```

> `companies` 和 `years` 从 chunks 表 distinct 获取，用于前端下拉框选项。`docs` 按 `doc_name` 去重聚合。

## 5. RAG管线详细设计

### 5.1 离线预处理脚本 ingest.py

输入：本地PDF目录 `backend/pdfs/`（6份年报平铺在同一目录）

处理流程：

1. 遍历目录下所有 PDF 文件。
2. 用 `parse_metadata` 从文件名提取公司和年份，通过硬编码映射表转为公司简称：

| 文件名提取结果 | 映射为简称 |
|---|---|
| `贵州茅台` | `茅台` |
| `宁德时代` | `宁德时代` |

3. 灌库幂等：插入前先删除同名年报的旧 Chunk：
```python
supabase.table('chunks').delete().eq('doc_name', filename).execute()
```

4. 用 pdfplumber 打开 PDF，遍历每页。
5. 提取文本：`page.extract_text()`，清洗页眉页脚（页眉为公司全称+年度报告标题，页脚为 `N/总页数` 或纯页码）。
6. 提取表格：`page.extract_tables()`，清洗后转为 Markdown，标记 `chunk_type='table'`。
7. 文本切片：使用 LangChain 的 `RecursiveCharacterTextSplitter`，`chunk_size=512`（字符），`overlap=50`（字符），分隔符 `['\n\n', '\n', '。', '；', '，', ' ']`。
8. 表格切片：若表格 Markdown 长度 > 1024 字符，按行切半（保留表头）。
9. 对每个 chunk 调用本地 `bge-large-zh` 生成 embedding（1024维）。模型路径硬编码，防止联网。
10. 分批插入 Supabase chunks 表（每批 100 条）。

**表格清洗要点**（基于贵州茅台、宁德时代 2025 年度报告实测）：

- **合并单元格**：pdfplumber 对合并单元格返回 `None`。宁德时代财务报表（9列宽表）中约40-50%的单元格为 None 分隔列，需自动过滤全 None 列。
- **长文本跨行**：宁德时代利润表中长科目名被拆为两行，需检测并合并。
- **多行表头**：茅台"本期比上年同期增减(%)"等表头跨2-3行，合并行后转 Markdown 即可，LLM 可从上下文理解列含义。
- **金额单位**：茅台用"元"（如 `168,838,102,514.79`），宁德时代部分报表用"千元"（如 `423,701,834`）。建议在表格 chunk 前附加页面上的单位说明行。

注意：运行前确保 torch 为 CUDA 版，利用 RTX 4060 加速。

### 5.2 在线问答流程（/api/chat）

1. 接收 `question`、`company`（可选）、`year`（可选）。
2. 通过远程 Embedding API（SiliconFlow，同一模型 `BAAI/bge-large-zh-v1.5`）将 `question` 编码为1024维向量。
3. 调用 `supabase.rpc('match_chunks', {...})`，传入过滤条件，返回 top-4。
4. 组装 Prompt：

```text
你是一个财报分析助手。请根据以下财报片段回答问题，若片段中无明确答案，请直接说"未在资料中找到"。
---片段开始---
{chunk1.content}
{chunk2.content}
...
---片段结束---
问题：{question}
回答（仅基于片段，不要外延）：
```

5. 调用 DeepSeek API（兼容 OpenAI 接口），获取回答。
6. 返回 `{answer, sources}`。

### 5.3 错误处理

任一环节失败时，后端返回 `{"error": "具体错误描述"}`，HTTP 状态码 200（业务错误不走 HTTP 500）。前端使用 `message.error()` 展示错误信息。

错误场景：

| 环节 | 错误描述示例 |
|------|-------------|
| Embedding API 调用失败 | "Embedding服务不可用，请稍后重试" |
| Supabase RPC 调用失败 | "数据库查询失败，请稍后重试" |
| LLM API 调用失败 | "AI服务暂时不可用，请稍后重试" |
| question 为空 | "问题不能为空" |
| /api/upload 文件缺失 | "请选择要上传的文件" |

## 6. 前端页面设计（Next.js Pages Router + AntD）

### 6.1 页面路由

- `/` → 问答页（核心），文件：`pages/index.tsx`
- `/manage` → 财报管理页，文件：`pages/manage.tsx`
- `/about` → 关于页，文件：`pages/about.tsx`

### 6.2 问答页组件

- `Select`（公司选择，默认"不限"） + `Select`（年份选择，默认"不限"）
- `Input.Search`（提问框）
- 空态：回答区和 sources 区隐藏，提问框在上方
- 加载态：回答区显示 `Spin`
- 结果态：`Card`（回答区） + `Collapse`（折叠面板，展示 sources 列表，含 doc_name、page、snippet）
- 错误态：`message.error()` 弹窗提示

公司/年份下拉框选项从 `/api/docs` 动态获取。

### 6.3 管理页组件

- `Upload`（拖拽上传 PDF，action="/api/upload"）
- `Table`（显示已入库年报列表，含公司、年份、chunk数量）

> 管理页仅展示 chunks 表中的已入库数据，不展示 Storage 中的暂存文件状态（上传与灌库分离）。

## 7. 环境变量（.env）

```env
# Supabase
SUPABASE_URL=your-project-url
SUPABASE_ANON_KEY=your-anon-key

# LLM（生成回答）
LLM_API_KEY=your-api-key
LLM_API_BASE=https://api.deepseek.com/v1
LLM_MODEL=deepseek-v4-flash

# Embedding（在线问答用，离线ingest直接用本地模型）
EMBEDDING_API_KEY=your-siliconflow-api-key
EMBEDDING_API_BASE=https://api.siliconflow.cn/v1
EMBEDDING_MODEL=BAAI/bge-large-zh-v1.5
```

> 注：离线预处理（ingest.py）在本地 GPU 环境运行，直接加载 `sentence-transformers` 模型，无需调用远程 Embedding API。远程 API 仅用于线上后端（Vercel Serverless），以规避模型包体积超限问题。

## 8. Golden Set 验收测试

| # | 问题 | 期望 |
|---|------|------|
| 1 | "茅台 2023 毛利率是多少？" | 返回具体数字（如 91.96%），sources 指向正确页码 |
| 2 | "宁德时代 2024 营收增速是多少？" | 返回具体数字，sources 指向正确页码 |
| 3 | "宁德时代和茅台 2023 年营收增速谁更高？" | 排序正确，跨公司比较准确 |

## 9. 部署架构

- **前端**：Vercel，前端通过 `NEXT_PUBLIC_API_URL` 环境变量指向后端地址
- **后端**：Vercel Serverless（`api/app.py`），Embedding 走远程 API，包体积 < 50MB
- **开发环境**：前端通过 `next.config.js` 的 `rewrites` 代理到 `localhost:5000`
