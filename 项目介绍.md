# 财报智能问答系统（RAG）— 项目介绍

## 项目概述

基于 **RAG（检索增强生成）** 技术的财报智能问答系统，用户通过自然语言提问，系统从 PDF 年报中检索相关片段，经精排后由大模型生成带引用的回答。

**数据范围**：贵州茅台、宁德时代 2023-2025 年度报告（6 份 PDF，共 5985 个文本片段）

**线上演示**：https://frontend-sigma-nine-90.vercel.app

---

## 核心亮点

### 1. 完整的 RAG 管线

```
用户提问 → Embedding → 向量检索(k=10) + 关键词补充
         → 去重 → bge-reranker-v2-m3 精排
         → 向量top-4 + 每公司关键词best-2 → Prompt → DeepSeek 生成
```

- **向量检索**：pgvector cosine 相似度，BAAI/bge-large-zh-v1.5（1024 维）
- **关键词补充**：针对财务表格数据（利润/现金流/分红等 9 类指标），LIKE 兜底检索，确保核心数据不遗漏
- **Rerank 精排**：bge-reranker-v2-m3 Cross-encoder 重排序，向量+关键词混合取 top-6~8 送入 LLM
- **防混淆**：每片段标注 `[公司 年份年报 第N页]`，跨公司比较不串数据

### 2. 多轮优化的检索策略

| 问题 | 初始方案 | 优化后 |
|------|---------|--------|
| 纯向量检索"利润" | 命中营收/毛利率页，利润数据 p6 排名 20+ | 关键词补充确保利润数据命中 |
| 跨公司"利润对比" | 一家公司数据被另一家挤占 | 每公司独立分配关键词额度 |
| "2023 到 2025 年"跨年 | 只提取第一个年份 | findall 提取所有年份 |
| Reranker 对表格评分低 | 关键词 Chunk 假值 1.0 排序 | 真实 Rerank 分数 + per-company best-2 |
| 回答拒答 | Prompt 过于严格（"无明确答案就说未找到"） | 放松为"理解片段后作答，无数据才说未找到" |

### 3. 完整的前后端

- **前端**：Next.js 14 + Ant Design + react-markdown（Markdown 渲染回答）
- **后端**：Flask（本地）+ Vercel Serverless（线上），3 个 REST API
- **数据库**：Supabase PostgreSQL + pgvector 向量索引
- **PDF 解析**：pdfplumber 正文提取 + 表格转 Markdown
- **全免费**：Embedding / Reranker 走 SiliconFlow 免费 API，LLM 走 DeepSeek 学生额度

## 演示用例（建议展示顺序）

### Case 1：单公司单指标
**"茅台 2023 年毛利率是多少？"**
→ 回答：92.11%，引用茅台 2023 年报第 14 页

### Case 2：跨公司对比
**"宁德时代和茅台 2024 年谁的利润更高？"**
→ 回答：茅台 862 亿 > 宁德时代 507 亿，分别引用双方年报

### Case 3：跨年趋势
**"宁德时代 2023 年到 2025 年经营现金流净额分别是多少？"**
→ 回答：928 亿 → 970 亿 → 1332 亿，逐年增长

### Case 4：分红方案
**"茅台 2023 年到 2025 年的分红方案分别是？"**
→ 回答：每 10 股 308.76 元 / 276.24 元 / 279.93 元

### Case 5：多公司多年度营收
**"宁德时代和茅台 2023-2025 年营业收入分别是多少？谁规模更大？"**
→ 宁德时代 4009 亿 > 茅台 1477 亿，差距逐年缩小

```
┌─────────────────────────────────────────────────┐
│  前端 Next.js 14 + AntD                          │
│  localhost:3000 / frontend-sigma-nine-90.vercel.app │
├─────────────────────────────────────────────────┤
│  后端 Flask / Vercel Serverless                  │
│  /api/chat  /api/upload  /api/docs               │
├─────────────────────────────────────────────────┤
│  Supabase PostgreSQL + pgvector                  │
│  chunks 表（5985 行，id/doc_name/company/year/    │
│  page/content/embedding）                        │
├─────────────────────────────────────────────────┤
│  外部 API                                        │
│  SiliconFlow: bge-large-zh-v1.5 (Embedding)     │
│  SiliconFlow: bge-reranker-v2-m3 (Rerank)       │
│  DeepSeek: deepseek-v4-flash (LLM)               │
└─────────────────────────────────────────────────┘
```

## 技术选型理由

| 选型 | 理由 |
|------|------|
| Supabase + pgvector | 免运维 BaaS，向量检索开箱即用，d3 课程推荐 |
| bge-large-zh-v1.5 | 中文语义理解优秀，1024 维，SiliconFlow 免费额度充足 |
| bge-reranker-v2-m3 | Cross-encoder 精排，有效过滤向量检索噪声 |
| DeepSeek | 1M 上下文窗口，中文财报理解好，学生免费 |
| pdfplumber | 表格抽取能力强，轻量无额外依赖 |
| Vercel | 前后端一键部署，免费额度够用 |
