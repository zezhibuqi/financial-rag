# CONTEXT — 财报智能问答系统

## 核心实体

### Chunk
从一份**年报**中切分出的最小检索单元。一个 Chunk 包含一段原文（文本或 Markdown 表格），对应到年报中的某一页。每个 Chunk 被向量化后存入 `chunks` 表，是 RAG 检索的基本单位。切片大小按字符数计（chunk_size=512，overlap=50），表格 Chunk 最多 1024 字符。

### 年报 (Annual Report)
一份 PDF 格式的上市公司年度报告。每份年报属于一家**公司**，对应一个**财务年度**。入库后，年报被解析为若干个 Chunk。系统目前支持 2 家公司各 3 个财务年度的年报（共 6 份）。

### 公司 (Company)
年报所属的上市公司。系统内部统一使用**公司简称**（如 `茅台`、`宁德时代`），不存储全称。简称在离线灌库时由文件名通过硬编码映射表解析得出：

| 文件名中提取 | 映射为简称 |
|---|---|
| `贵州茅台` | `茅台` |
| `宁德时代` | `宁德时代` |

用户查询时以简称为过滤条件。

### 财务年度 (Fiscal Year)
年报对应的时间跨度，用年份整数表示（如 `2023`）。当前数据覆盖 2023、2024、2025 三年。

### 灌库 (Ingestion)
将 PDF 年报解析为 Chunk、生成 Embedding 并写入数据库的**离线批处理操作**。灌库在本地 GPU 环境运行，与线上 `/api/upload` 完全分离。灌库对同一年报幂等（先删后插）。

### 上传 (Upload)
用户通过前端管理页将 PDF 年报上传至 Supabase Storage 的操作。上传**不触发**解析或灌库，仅做文件暂存。

## 关系

- 一份年报 ∈ 一家公司
- 一份年报 ∈ 一个财务年度
- 一份年报 → N 个 Chunk
- 一个 Chunk ∈ 一份年报

## 不使用的术语

- **文档 (Document)**：容易与"年报"混淆，系统内统一使用"年报"。
- **全文 (Full Text)**：年报的完整文本，不做为领域概念使用。
